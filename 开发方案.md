## 一、项目定位与核心目标

### 1.1 产品定位

TCli（Terminal Code Intelligence CLI）是一个面向**中文开发者**、以**Windows 一等公民**为差异化卖点的 AI 代码辅助命令行工具。

核心理念：

- **零依赖安装**：单一二进制文件，无需 Node/Python/Docker 运行时
- **多模型统一**：OpenAI / Anthropic / 本地模型 / 兼容接口统一管理
- **中文优先**：提示词模板、错误信息、文档全面中文化
- **Windows 原生**：PowerShell / CMD 完整支持，无需 WSL

### 1.2 竞品差异化对比

| 功能点           | Claude Code | Codex CLI    | Aider   | Goose    | **TCli**       |
| ---------------- | ----------- | ------------ | ------- | -------- | -------------------- |
| Windows 原生支持 | 差          | 差（无沙箱） | 差      | WSL 依赖 | **一等公民**   |
| 中文优化         | 无          | 无           | 无      | 无       | **全面中文化** |
| 多模型统一       | 仅 Claude   | 仅 OpenAI    | LiteLLM | 广泛     | **统一管理**   |
| Token 预算显示   | 无          | 无           | 无      | 无       | **实时显示**   |
| 单二进制分发     | 否          | 否           | 否      | 是       | **是**         |
| MCP 协议         | 是          | 是           | 否      | 是       | **是（后期）** |
| 本地 AST 分析    | 无          | 无           | 是      | 是       | **是（后期）** |
| 启动动画         | 无          | 无           | 无      | 无       | **品牌化动画** |

### 2.3 AI 客户端统一接口设计

```go
// internal/ai/client.go

// Provider 枚举
type Provider string
const (
    ProviderOpenAI     Provider = "openai"
    ProviderAnthropic  Provider = "anthropic"
    ProviderCompatible Provider = "compatible"  // Deepseek, Qwen, Ollama 等
)

// 统一请求结构
type Request struct {
    Messages    []Message
    MaxTokens   int
    Temperature float32
    Stream      bool
}

// 统一流式回调
type StreamHandler func(delta string, done bool, err error)

// 统一客户端接口
type Client interface {
    Complete(ctx context.Context, req Request) (string, error)
    Stream(ctx context.Context, req Request, handler StreamHandler) error
    CountTokens(messages []Message) int
    ModelName() string
}
```

## 三、核心功能设计

### 3.1 命令列表（MVP）

#### `tcil chat` — 交互式对话（主打功能）

```
功能：
- Bubbletea 驱动的 TUI 界面
- 实时流式输出 AI 响应
- 底部状态栏显示：已用 token / 模型名称 / 当前 provider
- 支持多行输入（Shift+Enter 换行，Enter 发送）
- 历史记录：方向键上下翻历史
- 自动注入当前目录的代码上下文（可 --no-context 关闭）
- /clear  清空对话历史
- /copy   复制最后一条回复到剪贴板
- /model  切换模型
- /save   保存对话到文件

UI 布局：
┌─────────────────────────────────────────┐
│  TCli  claude-3-7-sonnet  ● Anthropic   │  ← 顶部标题栏
├─────────────────────────────────────────┤
│                                         │
│  [User] 帮我解释这段代码的作用           │
│                                         │
│  [Claude] 这段代码实现了...             │  ← Glamour 渲染 Markdown
│    ```go                                │
│    func main() { ... }                  │
│    ```                                  │
│                                         │
├─────────────────────────────────────────┤
│  > 在这里输入...                        │  ← 输入框
├─────────────────────────────────────────┤
│  tokens: 1,234/128k  ↑↓历史  /help     │  ← 状态栏
└─────────────────────────────────────────┘
```

#### `tcil ask "问题"` — 单次问答

```bash
# 基本用法
tcil ask "这个函数是做什么的？" --file main.go

# 管道支持（差异化功能，类似 ForgeCode）
cat error.log | tcil ask "这个报错什么原因"
git diff HEAD~1 | tcil ask "帮我写 commit message"

# 输出格式控制
tcil ask "解释这段代码" --file main.go --raw    # 纯文本，适合管道
tcil ask "解释这段代码" --file main.go --md     # Markdown 渲染
```

#### `tcil review <file>` — 代码审查

```bash
tcil review main.go                    # 单文件审查
tcil review ./src/                     # 目录审查
tcil review --staged                   # 审查 git staged 的文件
tcil review --diff HEAD~1              # 审查最近一次提交的改动

# 输出：结构化的审查报告（Glamour 渲染）
# - 潜在 Bug
# - 性能问题
# - 安全问题
# - 可读性建议
```

#### `tcil commit` — 智能 commit message

```bash
tcil commit                    # 分析 staged 文件，生成 commit message
tcil commit --push             # 生成后自动 push
tcil commit --convention       # 强制 Conventional Commits 格式
tcil commit --lang en          # 英文 commit（默认中文）
```

#### `tcil fix` — 修复代码错误

```bash
# 将编译错误通过管道传入
go build 2>&1 | tcil fix

# 直接指定错误信息
tcil fix --error "undefined: foo" --file main.go
```

#### `tcil config` — 配置管理

```bash
tcil config set openai.api_key sk-xxx
tcil config set default.provider anthropic
tcil config set default.model claude-3-7-sonnet
tcil config list                        # 显示所有配置（隐藏敏感值）
tcil config providers                   # 显示已配置的 provider 列表
```

### 3.3 Token 预算系统（差异化核心功能）

```
发送请求前：
1. 计算系统 prompt token 数
2. 计算注入的代码上下文 token 数
3. 计算对话历史 token 数
4. 总和接近模型上下文限制时：
   a. 先截断最早的对话历史
   b. 再压缩代码上下文（只保留函数签名）
   c. 显示警告："上下文已压缩，历史对话已截断 N 条"
5. 底部状态栏实时显示：已用 1,234 / 128,000 tokens
```

### 3.4 智能上下文收集

```
执行 tcil 命令时，自动收集：

1. 项目类型识别（检测 go.mod/package.json/Cargo.toml/requirements.txt）
2. git 信息：
   - 当前分支名
   - 最近 3 条 commit message
   - git status（未提交的文件列表）
3. 相关文件内容（按 token 预算动态调整数量）
4. 如有 .tcil/config.yaml，额外注入 extra_files
```

### 5.4 Bubbletea 动画实现代码

```go
// internal/tui/splash/model.go

package splash

import (
    "strings"
    "time"

    tea "charm.land/bubbletea/v2"
    "github.com/charmbracelet/lipgloss/v2"
)

// 动画阶段
type phase int
const (
    phaseTyping  phase = iota // 逐字打印 "TCli"
    phaseLogo                 // 展开完整 Logo
    phaseInfo                 // 显示版本 + 加载点
    phaseDone                 // 动画结束，通知父级
)

// 内部 tick 消息
type tickMsg time.Time

// 动画结束消息（通知根 Model 切换到主界面）
type DoneMsg struct{}

const logo = `
████████╗ ██████╗██╗     ██╗
   ██╔══╝██╔════╝██║     ██║
   ██║   ██║     ██║     ██║
   ██║   ██║     ██║     ██║
   ██║   ╚██████╗███████╗██║
   ╚═╝    ╚═════╝╚══════╝╚═╝`

const subtitle = "Terminal Code Intelligence CLI"

type Model struct {
    phase       phase
    typedChars  int        // 已打印的字符数（阶段一）
    dotCount    int        // 加载点数量（阶段三）
    width       int        // 终端宽度
    version     string     // 版本号，如 "v0.1.0"
    modelName   string     // 当前 AI 模型名
}

func New(width int, version, modelName string) Model {
    return Model{
        phase:     phaseTyping,
        width:     width,
        version:   version,
        modelName: modelName,
    }
}

func (m Model) Init() (tea.Model, tea.Cmd) {
    return m, tick(80 * time.Millisecond) // 每 80ms 触发一帧
}

func tick(d time.Duration) tea.Cmd {
    return tea.Tick(d, func(t time.Time) tea.Msg {
        return tickMsg(t)
    })
}

func (m Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    switch msg.(type) {
    case tickMsg:
        switch m.phase {
        case phaseTyping:
            name := "TCli"
            if m.typedChars < len(name) {
                m.typedChars++
                return m, tick(100 * time.Millisecond)
            }
            // 打完后停顿一下再展开 Logo
            m.phase = phaseLogo
            return m, tick(300 * time.Millisecond)

        case phaseLogo:
            m.phase = phaseInfo
            return m, tick(500 * time.Millisecond)

        case phaseInfo:
            if m.dotCount < 3 {
                m.dotCount++
                return m, tick(180 * time.Millisecond)
            }
            // 加载点打完，动画结束
            m.phase = phaseDone
            return m, func() tea.Msg { return DoneMsg{} }
        }
    }
    return m, nil
}

func (m Model) View() string {
    var sb strings.Builder

    // 渐变色：从紫色到青色
    gradient := []lipgloss.Color{
        "#BF5AF2", "#9B59B6", "#7D56F4", "#5DADE2", "#48C9B0",
    }

    centerStyle := lipgloss.NewStyle().Width(m.width).Align(lipgloss.Center)
    dimStyle    := lipgloss.NewStyle().Foreground(lipgloss.Color("#555555"))
    boldStyle   := lipgloss.NewStyle().Bold(true)

    switch m.phase {

    case phaseTyping:
        // 仅显示正在打印的文字，居中
        name := "TCli"[:m.typedChars]
        typed := lipgloss.NewStyle().
            Foreground(gradient[0]).
            Bold(true).
            Render(name)
        sb.WriteString(centerStyle.Render(typed))

    case phaseLogo, phaseInfo:
        // 用渐变色逐行渲染 Logo
        lines := strings.Split(strings.TrimPrefix(logo, "\n"), "\n")
        var logoLines []string
        for i, line := range lines {
            colorIdx := (i * len(gradient)) / len(lines)
            colored := lipgloss.NewStyle().
                Foreground(gradient[colorIdx]).
                Bold(true).
                Render(line)
            logoLines = append(logoLines, centerStyle.Render(colored))
        }
        sb.WriteString(strings.Join(logoLines, "\n"))
        sb.WriteString("\n\n")
        sb.WriteString(centerStyle.Render(dimStyle.Render(subtitle)))

        if m.phase == phaseInfo {
            sb.WriteString("\n")
            dots := strings.Repeat("●", m.dotCount) +
                strings.Repeat("○", 3-m.dotCount)
            info := boldStyle.Render(m.version) + "  ·  " +
                dimStyle.Render(m.modelName) + "  ·  初始化中 " +
                lipgloss.NewStyle().Foreground(gradient[2]).Render(dots)
            sb.WriteString(centerStyle.Render(info))
        }
    }

    return sb.String()
}
```

// main.go 中判断是否显示动画

func shouldShowSplash() bool {
    // 1. 非交互终端（管道/重定向），跳过
    if !isatty.IsTerminal(os.Stdout.Fd()) {
        return false
    }
    // 2. CI 环境，跳过
    if os.Getenv("CI") != "" {
        return false
    }
    // 3. 用户主动禁用
    if os.Getenv("TCLI_NO_SPLASH") != "" {
        return false
    }
    // 4. 单次问答命令（ask/commit/fix），跳过（速度优先）
    // 只有 tcli chat / tcli review 等 TUI 命令才显示
    return true
}

        // 在 goroutine 里调用 AI API

        // ...
    }

// Update 中处理流式消息
func (m Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    switch msg := msg.(type) {
    case StreamDeltaMsg:
        m.streamBuf.WriteString(msg.text)
        m.refreshViewport()
        return m, waitForNextDelta(deltaChan) // 继续监听下一个 delta

## 七、Windows 兼容性专项方案

这是差异化核心，需要专门处理：

| 问题            | 方案                                                           |
| --------------- | -------------------------------------------------------------- |
| ANSI 颜色不显示 | crossterm/lipgloss 自动检测 Windows Console，降级为基础颜色    |
| 路径分隔符      | 全程使用 `filepath.Join`，禁止硬编码 `/`                   |
| 配置路径        | 使用 `os.UserConfigDir()` 而非 `~`（返回 `%APPDATA%`）   |
| PowerShell 补全 | Cobra 原生支持 PowerShell 补全，`tcil completion powershell` |
| 终端检测        | 检测 `WT_SESSION` 环境变量判断是否 Windows Terminal          |
| 换行符          | 配置文件读写统一用 `\n`，输出时按平台适配                    |
